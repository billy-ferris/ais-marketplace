# Phase 02.4 Plan 03: Seed Production Guard + Upload Hardening Summary

Server-side hardening of two trust boundaries: a hard `NODE_ENV=production` refusal
(exit 1, no override) as the first statement of the destructive seed script (D-08),
and a server-authoritative R2 upload policy that allowlists image MIME types
(jpeg/png/webp â€” no SVG/GIF) and caps size at 5 MB via `createPresignedPost`'s
`content-length-range` condition (D-06/D-07).

## What Was Built

### Task 1 â€” `assertNotProduction()` seed guard (D-08)
- Exported pure `assertNotProduction()` in `apps/api/src/db/seed.ts`: logs a clear
  `console.error` and calls `process.exit(1)` when `NODE_ENV === 'production'`,
  before any `db.delete(...)` or Clerk user deletion. No override/escape-hatch env var.
- Called as the first statement of `main()`.
- Prerequisite refactor: `main()` now only auto-runs when `seed.ts` is the process
  entry point (`isDirectRun()` compares `process.argv[1]` to the module URL). Without
  this, importing the module for the unit test would execute the destructive seed and
  a `process.exit` that kills the test runner (the existing `categories.test.ts` avoids
  importing `seed.ts` for exactly this reason).
- `apps/api/src/__tests__/seed/guard.test.ts` (new): asserts exit(1) on production,
  no-op on development/test/unset, and that no "force" env var bypasses the guard.

### Task 2 â€” Upload content-type allowlist + 5 MB size cap (D-06/D-07)
- `ALLOWED_CONTENT_TYPES = ['image/jpeg', 'image/png', 'image/webp']`; non-allowlisted
  `contentType` is rejected with `400 { error: 'Unsupported content type' }` before any
  presign. SVG (stored-XSS vector) and GIF are intentionally excluded.
- Switched from presigned PUT (`getSignedUrl` + `PutObjectCommand`) to
  `createPresignedPost` with `Conditions: [['content-length-range', 1, 5242880],
  ['eq', '$Content-Type', contentType]]`. Response shape changed from `{ uploadUrl }`
  to `{ url, fields, publicUrl, key }`.
- `apps/api/src/__tests__/routes/uploads.test.ts` reworked to mock
  `@aws-sdk/s3-presigned-post`, assert allowlist rejections (no presign issued), assert
  the 5 MB `content-length-range` condition, and assert the new response shape.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Seed module was not importable for unit testing**
- **Found during:** Task 1
- **Issue:** `seed.ts` calls `main().then(process.exit)` at module top level, so importing
  `assertNotProduction` in a test would run the destructive seed and exit the runner.
- **Fix:** Wrapped the `main()` invocation in an `isDirectRun()` entry-point guard.
- **Files modified:** apps/api/src/db/seed.ts
- **Commit:** 95ed201

**2. [Rule 3 - Blocking] tsc type-skew between AWS SDK packages**
- **Found during:** Task 2 (build verification)
- **Issue:** `@aws-sdk/client-s3@^3.1004.0` pulled `@smithy/types@4.13.0` while the new
  `@aws-sdk/s3-presigned-post@3.1077.0` pulled `4.15.0`; the `s3Client` was not assignable
  at the `createPresignedPost` call, failing `tsc`.
- **Fix:** Bumped `@aws-sdk/client-s3` to `^3.1077.0` (same release train) to dedupe
  `@smithy/types`. Also removed the now-unused `@aws-sdk/s3-request-presigner`.
- **Files modified:** apps/api/package.json, pnpm-lock.yaml
- **Commit:** 7db5981

**3. [Rule 3 - Blocking] guard test process.exit spy typing failed tsc**
- **Found during:** Task 2 (build verification)
- **Issue:** `process.exit`'s `(code?) => never` signature is not assignable to the generic
  `ReturnType<typeof vi.spyOn>`, breaking the `tsc` build.
- **Fix:** Loosened the spy variable typing in guard.test.ts.
- **Files modified:** apps/api/src/__tests__/seed/guard.test.ts
- **Commit:** 5127065
