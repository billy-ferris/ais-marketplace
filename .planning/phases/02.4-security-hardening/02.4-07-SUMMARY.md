# Phase 02.4 Plan 07: R2 Presigned-PUT Revert Summary

Reverted the upload transport from S3 presigned POST back to presigned PUT (R2 has no presigned POST support) while keeping the D-06 content-type allowlist and the D-07 5 MB cap intact via a signed exact ContentLength.

## What Was Built

- **API presign endpoint (`apps/api/src/routes/uploads.ts`):** Swapped `createPresignedPost` for `getSignedUrl` over a `PutObjectCommand`. The command pins `ContentType` and an exact `ContentLength` so R2 rejects a mismatched upload at ingest. The handler now reads `fileSize`, rejects missing/non-numeric/`<= 0` values with 400, keeps the D-06 allowlist rejection, and adds a D-07 `fileSize > 5 MB` rejection — all before signing. Response shape is now `{ uploadUrl, publicUrl, key }`.
- **API unit tests (`apps/api/src/__tests__/routes/uploads.test.ts`):** Rewrote mocks to `getSignedUrl` + a `PutObjectCommand` constructor that records its input. Asserts the PUT response shape, allowlist rejection (no sign), accept cases (signed once), a new `>5 MB` rejection, a boundary-accept at exactly 5 MB, and that the signed command carries the correct `ContentType`/`ContentLength`. 13 tests pass.
- **Client hook (`apps/web/src/hooks/useUpload.ts`):** Replaced the multipart/FormData POST with a raw-body `fetch(uploadUrl, { method: 'PUT', body: file, headers: { 'Content-Type': file.type } })`. Sends `fileSize: file.size` for the server D-07 gate, preserves the WR-02 `!uploadRes.ok` throw, and keeps the exported `{ uploadFile, isUploading }` signature stable for Plan 08.
- **Dependency swap:** `@aws-sdk/s3-presigned-post` removed, `@aws-sdk/s3-request-presigner ^3.1077.0` added (official `@aws-sdk` sibling of the already-trusted `client-s3`; RESEARCH audit = Approved). `package.json` and `pnpm-lock.yaml` updated together.
