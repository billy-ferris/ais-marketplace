### `apps/api/src/routes/users.ts` — GET `/:id` self-or-admin, deny-by-404 (D-01/D-02)

**Analog B — admin gating already exists** (`GET /`, L36): `requireAuth(), requireRole('admin')`. But D-01 forbids admin-only (users must read their own record), so the gate goes *inside* the handler, not as `requireRole` middleware.

**Current target handler** (L47-70) — the excerpt to modify:
```typescript
// GET /:id - Get user by ID with company relation (any authenticated user)

  try {
    const id = Number(req.params.id);
    if (Number.isNaN(id)) {
      res.status(400).json({ error: 'Invalid user ID' });
      return;
    }

    const user = await db.query.users.findFirst({
      where: eq(users.id, id),
      with: { company: true },
    });

    if (!user) {                                          // <-- extend this deny
      res.status(404).json({ error: 'User not found' });
      return;
    }

    res.json(user);
```

**How to modify (copy `getCurrentUser` import — already imported L5):**
```typescript
const { userId, role } = getCurrentUser(req);
// ...findFirst as above...
// SINGLE deny path covers "missing" AND "found but not yours" — no enumeration oracle (D-02)
if (!user || (role !== 'admin' && user.clerkId !== userId)) {
  res.status(404).json({ error: 'User not found' });
  return;

```
- `getCurrentUser` is already imported: `import { requireAuth, requireRole, getCurrentUser } from '../middleware/auth.js';` (L5).
- `userId` is the **Clerk id string**; compare against `user.clerkId` (NOT `user.id`). The `GET /me` sibling proves this pattern (`eq(users.clerkId, userId)`, L20).
- **Pitfall:** do NOT split into a separate 403 branch — that reintroduces the enumeration oracle D-02 closes.

**No existing escaper** — add two dependency-free helpers (RESEARCH Pattern 5). The sinks are all in this file:

**Sink 3 — subject line** (`SUBJECT_MAP`, L28-32), consumed at L125 `SUBJECT_MAP[params.type](params.listingName)`:

**Helpers to add (from 02.1-REVIEW CR-02, exact):**
```typescript
function escapeHtml(s: string): string {
  return s.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;')
          .replace(/"/g, '&quot;').replace(/'/g, '&#39;');
}
function sanitizeHeaderValue(s: string): string {
  return s.replace(/[\r\n]/g, ' ');
}
```

- Resend (`resend.emails.send`, L128-133) sends `html` verbatim — no auto-escaping.

## Shared Patterns

### Auth-context reads (applies to users.ts, companies.ts, listings.ts)
**Source:** `apps/api/src/middleware/auth.ts`
```typescript
getCurrentUser(req)  // → { userId: clerkId|null (STRING), role: UserRole|null, sessionClaims }   (L49-56, no DB)
getCompanyUser(req)  // async → { id, companyId, role, email, firstName } | null   (L63-77, DB lookup)
```
- `userId` is the **Clerk id string** — compare against a fetched local user's `clerkId`, never `users.id`.
- `role` comes from Clerk session claims (no DB round-trip).
- `companyId` is ONLY available via the async `getCompanyUser(req)` (not in `getCurrentUser`). Companies redaction (D-03) and PATCH brandId (D-12) both need it.
- `requireAuth()` (L15-24, 401 JSON) and `requireRole(...roles)` (L31-43, 403 JSON) stay as-is — the phase layers handler-level checks on top; do NOT replace them.

### Handler error convention (all route files)
Every handler is `async (req, res, next)` with `try { ... } catch (err) { next(err); }` and early-returns on deny (`res.status(...).json({ error }); return;`). New deny branches MUST follow the same early-return shape (no thrown errors for expected denies).

### Deny-by-404 for reads (D-02, users + companies)
Return **404**, never 403, on an authorization denial for a read, to prevent id enumeration. Single deny path covers both "missing" and "not yours."

### Drizzle imports convention
- `isNull`, `sql`, `eq`, `and` → from `drizzle-orm` (root). Existing code uses root (`middleware/auth.ts` L4, `routes/*`).
- `uniqueIndex`, `pgTable`, column builders → from `drizzle-orm/pg-core`.
- Match the root-export convention for `isNull` even though the 02-REVIEW snippet imported it from `pg-core` (both re-export).

### Migration workflow (D-09/D-10/D-13)
Project runs **`db:push`** (`pnpm --filter @ais/api db:push`) — there is **no `drizzle/` migrations directory**. Do not introduce generated-migration files. Push prints SQL for the constraint drop/create; dedup data (D-10) BEFORE push.
