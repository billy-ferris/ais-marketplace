|----------|-------------|
| browser client → API presign endpoint | Untrusted fileName/contentType/fileSize cross here; server is the authoritative validator before any URL is signed |
| API-signed URL → R2 (PutObject) | Signed ContentType/ContentLength constrain what R2 will accept at ingest |
| pnpm dependency graph | Supply-chain surface for the AWS SDK swap |

## STRIDE Threat Register (ASVS L1, block on high)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-024-01 | Tampering/Elevation | uploads.ts presign (D-06) | mitigate | Reject contentType outside image/jpeg,png,webp with 400 before signing; pin ContentType on PutObjectCommand so R2 rejects a mismatched upload (blocks stored-XSS SVG) |
| T-024-02 | Denial of Service | uploads.ts presign (D-07) | mitigate | Reject client fileSize > 5 MB with 400 before signing; sign exact ContentLength so R2 rejects at ingest |
| T-024-03 | Spoofing | client-declared fileSize | mitigate | Exact signed ContentLength forces actual bytes to equal the declared size or R2 refuses; residual exact-length fragility accepted per RESEARCH (honest ≤5 MB uploads succeed) |
| T-024-04 | Info Disclosure | useUpload.ts (WR-02) | mitigate | Check `uploadRes.ok` before returning publicUrl so a failed/absent object is never surfaced as a valid URL |
