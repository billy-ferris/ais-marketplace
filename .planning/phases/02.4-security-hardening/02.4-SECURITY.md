## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Browser → API (authenticated) | Clerk-authenticated SPA calls the Express API | User/company PII, listing tampering payloads, brandId reassignment |
| API → Postgres | Drizzle ORM writes; DB-level constraints are the last line of integrity | Slugs, primary-image flags, FK relationships |
| API → Resend (email) | User-controlled listing names / rejection reasons flow into HTML body + subject header | XSS/HTML sinks, CRLF header injection |
| Browser → R2 (direct upload) | Client uploads directly to R2 via a server-signed presigned PUT | File content-type + size (server-signed policy is authoritative) |
| Client route layer | React Router `/manage/*` UX gating (non-authoritative; server API remains the enforcement point) | Role-based page access |
| Seed script → environment | Destructive seed script guarded against production | NODE_ENV production refusal |
