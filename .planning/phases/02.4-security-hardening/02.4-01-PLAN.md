<objective>
Close the two Phase 01 IDOR criticals: broken access control on `GET /users/:id` (any authenticated user could read any user record) and PII/commercial-data exposure on `GET /companies` (marginPercentage + contact fields returned to everyone).

Purpose: The server is the real authorization boundary. These handler-level gaps leak other users' records and manufacturers' margin/PII to any authenticated caller. This plan enforces self-or-admin read scope with anti-enumeration deny-by-404, and field-level redaction for company reads.

Output: Hardened `users.ts` and `companies.ts` handlers, plus new Vitest coverage proving each authorization branch.
</objective>
