# Phase 02.4 Plan 08: Surface Upload Rejections (Gap 2 / UAT 3) Summary

Closes Gap 2 from 02.4-06-UAT: server-rejected uploads (D-06 SVG → 400, oversize) were swallowed by empty `catch {}` blocks and only logged to console. `ImageUploader` now accepts an external `error` prop and the three consuming forms resolve and display the rejection message.

## What Was Built

**Task 1 — ImageUploader external error path** (`b4926cc`)
- Added optional `error?: string` prop to `ImageUploaderProps`, rendered in the existing `text-destructive` `<p>`.
- Renamed the internal size-precheck state to `internalError` and compute `displayError = error ?? internalError`, so an external server rejection takes precedence but the internal size pre-check still renders when no external error is set (no regression).
- Added a UX-only client content-type pre-check inside `handleFile` mirroring the D-06 allowlist (`image/jpeg`, `image/png`, `image/webp`): a disallowed `file.type` sets the internal error and returns before `onChange`. This is fast feedback only — the server presign (Plan 07) remains authoritative.

**Task 2 — Wire the three consuming forms** (`996adcb`)
- **BrandDialog.tsx** and **ListingForm.tsx** (both render `ImageUploader`): added a local `uploadError` state, cleared at the start of `handleImageUpload`, set in the catch via `getApiErrorMessage(err, 'Upload failed')`, and passed `error={uploadError ?? undefined}` to `<ImageUploader>`.
- **SkuInlineEditor.tsx** (raw `<input type="file">` cell, no `ImageUploader`): the rejection is written into the existing per-cell `skuErrors` map keyed `${index}:imageUrl` (cleared at handler start). Because the image cell is a raw button not covered by `renderRequiredCell`, the message renders inline below the cell in the existing `text-destructive` style.
- Removed all empty `catch {}` blocks around `await uploadFile(...)` in the three forms; all use `getApiErrorMessage` (not bare `err.message`) so `ApiError` field/form messages resolve correctly.
