<objective>
Wire the currently-dead client `RoleGuard` into the router per the D-04 route->role map with `ForbiddenPage` as the deny UX (D-04/D-05), and update the upload hook to consume the new presigned-POST contract introduced in Plan 03 (D-07 client ripple).

Purpose: `RoleGuard` is built and functional but never mounted — every `/manage/*` route is currently reachable by any logged-in role (defense-in-depth gap; the API is still the real boundary). Separately, Plan 03 changes the presign response from `{ uploadUrl }` (PUT) to `{ url, fields }` (POST), so the client upload path must switch to a multipart POST or uploads break.

Output: Router with per-route RoleGuard wrapping and an updated `useUpload` hook.

NOTE: apps/web has NO test harness (project-wide practice). Both changes are verified by manual browser UAT in Plan 06 — do NOT create React/UI tests here. Verification for these tasks is type-check/build only.
