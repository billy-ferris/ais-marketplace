## Read-access authorization policy

### GET /users/:id scope
| Option | Description | Selected |
|--------|-------------|----------|
| Self-or-admin | User reads own record; admins read anyone | ✓ |
| Admin-only | Only admins can hit the endpoint | |

### Deny response code (users + companies)
| Option | Description | Selected |
|--------|-------------|----------|
| 404 Not Found | Hides existence, prevents id enumeration | ✓ |
| 403 Forbidden | Explicit but leaks that id is valid | |

### GET /companies non-owner handling
| Option | Description | Selected |
|--------|-------------|----------|
| Field-redact | Keep readable, strip marginPercentage + contact PII for non-admin/non-owner | ✓ |
| Admin/owner-only endpoint | Block entirely unless admin or owning company | |

**User's choice:** Self-or-admin; 404 on deny; field-redaction for companies.
**Notes:** Redaction chosen so retailers can still browse manufacturer companies/brands. 404 is a deliberate anti-enumeration choice applied to both users and companies denies.
