## Decisions Made
- **ApprovalReviewPage as dedicated page**: Instead of inline approve/reject buttons in the table, admin navigates to a detail page showing full listing info before making a decision. Better UX for reviewing complex listings.
- **ListingViewPage for active listings**: Active listings show View (read-only detail) instead of Edit, since editing active listings could disrupt marketplace.
- **Admin notification routing**: Admin clicking a "listing_submitted" notification routes to `/manage/approvals/:id` review page instead of the listing edit page.
- **Categories API read access**: Opened GET endpoints to manufacturer and retailer roles so manufacturers can assign categories when creating listings.
- **Email service lazy init**: Avoid instantiating Resend client when no API key is set, preventing startup crashes in dev.
