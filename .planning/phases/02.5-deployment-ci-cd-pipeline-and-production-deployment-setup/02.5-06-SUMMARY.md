# Phase 02.5 Plan 06: Vercel SPA Hosting (Config Artifacts) Summary

Authored and committed the frontend deploy config for the split-host topology: a minimal `vercel.json` catch-all rewrite (`/(.*)` → `/index.html`) enabling client-side deep-link routing, plus an annotated `apps/web/.env.example` documenting the two build-time `VITE_*` variables and their production values. Per the user's "artifacts now, provision later" decision, no live Vercel provisioning was performed — every provisioning step is captured verbatim below for a later human pass.

## What Was Built

**`vercel.json`** (repo root, new):
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```
- Catch-all rewrite so any route (e.g. `/manage/approvals`) deep-links to the SPA shell (`index.html`) instead of 404ing — required for `react-router` v7 client-side routing (RESEARCH Pattern 5 / PATTERNS.md line 95).
- Deliberately minimal: Root Directory (`apps/web`), framework preset (Vite), and pnpm detection are set in the Vercel UI, not in committed config.

**`apps/web/.env.example`** (modified):
- Retained the existing 2-key contract — no new keys added.
- Annotated dev vs prod values: `VITE_CLERK_PUBLISHABLE_KEY` → `pk_test_*` (dev) / `pk_live_*` (prod, from the Plan 07 Clerk production instance); `VITE_API_BASE_URL` → `http://localhost:3001` (dev) / the Railway API URL (prod, from Plan 05).
- Documented that both are Vite build-time vars injected from the Vercel env store (D-07/D-08), never committed with real values, and that only the *publishable* (public) Clerk key belongs in the web app — no secret key (mitigates threat T-025-06-I).

## Deferred Human Provisioning Steps

These are the live-provisioning steps from Task 2 that were NOT executed. Run them in the Vercel dashboard when ready to provision the frontend host. The committed `vercel.json` is read automatically by Vercel on deploy.

1. **Import the repo & set Root Directory.**
   - In Vercel: import the GitHub repo `ais-marketplace`.
   - Set **Root Directory = `apps/web`**.
   - Framework preset: **Vite**.

2. **Confirm pnpm (not npm) is used for the build (Pitfall 6).**
   - The build log MUST show **pnpm**, resolving `@ais/shared` (`workspace:*`) from the root `pnpm-lock.yaml` and honoring the `packageManager: pnpm@9.15.0` pin in root `package.json`.
   - If Vercel mis-detects npm, override the Install Command to run at the workspace root, e.g.:
     ```
     pnpm install --frozen-lockfile
     ```
     (run from the monorepo root so the workspace + lockfile resolve).
   - Build Command: default Vite build (`pnpm build` → `tsc -b && vite build`). Output Directory: `dist`.

3. **Set build-time environment variables (D-08, host-native env store — injected at build time only):**
   - `VITE_API_BASE_URL` = the Railway API URL (from Plan 05), e.g. `https://<service>.up.railway.app`
   - `VITE_CLERK_PUBLISHABLE_KEY` = the Clerk publishable key. Use `pk_live_*` once the Plan 07 production Clerk instance exists; `pk_test_*` for preview/staging.
   - Do NOT add any Clerk *secret* key — the web bundle is public.

4. **Enable / confirm PR preview deploys (default on).**
   - Open a PR and confirm a preview URL builds and serves the SPA.
   - Visit a deep-link route directly on the preview (e.g. `/manage/approvals`) and confirm it returns the SPA (200 HTML), NOT a 404 — proves the `vercel.json` rewrite is active.

5. **Confirm the production deployment** serves the SPA at the production domain.

6. **Cross-plan wiring (Plan 07):** once the Vercel production/preview origins exist, add them to the API CORS allowlist (Plan 04 provides the allowlist mechanism; Plan 07 wires the final origin values).

**Resume signal for the checkpoint (when a human runs the above):** Type "approved" once the Vercel project is live (pnpm build, SPA deep-links work, PR preview builds) — or describe any build-log issue.
