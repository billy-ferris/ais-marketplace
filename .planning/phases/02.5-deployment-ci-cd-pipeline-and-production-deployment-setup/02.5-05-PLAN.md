## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deploy lifecycle → prod DB | Migrations mutate the production schema |
| host env store → runtime | Secrets injected into the API container |
| prod runtime config | NODE_ENV / TLS-verification flags govern safety guards |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-025-05-T | Tampering / Integrity | serving an un-migrated DB | mitigate | `preDeployCommand` migrate runs in a separate container before serve; non-zero exit aborts the deploy (D-06). |
| T-025-05-E | Elevation / Info Disclosure | dev-only TLS-reject leaking to prod | mitigate | `NODE_TLS_REJECT_UNAUTHORIZED` must be ABSENT from Railway env; real provider certs verify (D-08). Operator checklist confirms. |
| T-025-05-T2 | Tampering | destructive seed/push in prod | mitigate | `NODE_ENV=production` activates the `assertNotProduction()` guard; pre-deploy runs migrate only, never seed/push (D-05/D-08). |
