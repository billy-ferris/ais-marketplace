# Phase 02.5 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest (API suite, already present) + shell/CLI assertions for infra config |
| **Config file** | `apps/api/vitest.config.ts` (present); eslint/typecheck configs — **none, Wave 0 installs** |
| **Quick run command** | `pnpm --filter @ais/api test` |
| **Full suite command** | `pnpm turbo run lint typecheck test` |
| **Estimated runtime** | ~5 seconds (171 API tests ~2s + lint/typecheck) |

> **Nyquist note (infra phase):** This phase produces mostly config/pipeline artifacts, not product logic. "Validation" here means: (a) the CI gate actually runs the intended checks and fails red on a broken commit, (b) the production `start` command boots and serves `/api/health`, (c) generated migrations apply cleanly against a fresh DB, and (d) CORS/Clerk allowlist admits the Vercel origin and rejects others. These are verified by executing the commands/config, not by unit tests.

---

## Sampling Rate

- **After every task commit:** Run `{quick run command}` relevant to the task (e.g. `pnpm --filter @ais/api lint` after adding eslint; boot the API after touching the start script).
- **After every plan wave:** Run `pnpm turbo run lint typecheck test`.
- **Before `/gsd:verify-work`:** Full suite green + production `start` boots locally + `drizzle-kit migrate` applies against a scratch DB.
- **Max feedback latency:** ~10 seconds.

> **Checkpoint exemption (infra phase):** `checkpoint:human-action` / `checkpoint:human-verify` tasks — Railway/Vercel/Clerk/GitHub dashboard provisioning (Plans 05, 06, 07 and the branch-protection step in Plan 03) — are genuinely un-automatable and are **exempt from the "no 3 consecutive tasks without an automated verify" density rule (Dimension 8c)**. Their proof lives in the "Manual-Only Verifications" table below and in each task's `<how-to-verify>` / `<resume-signal>`. Do not re-litigate this during execution or verify.
