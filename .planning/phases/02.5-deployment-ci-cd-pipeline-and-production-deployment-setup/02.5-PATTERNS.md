### `.github/workflows/ci.yml` (config, event-driven) — GREENFIELD

**Analog:** none in repo (no `.github/` directory exists — verified). Copy the authoritative shape from `02.5-RESEARCH.md` Pattern 1 (adapted from turborepo.dev GitHub Actions guide).

**Repo facts the workflow must honor (verified in this repo):**
- `pnpm@9.15.0` pin lives in root `package.json` line 15 (`"packageManager": "pnpm@9.15.0"`) → use `pnpm/action-setup@v4` with **no** hardcoded `version:` (reads the pin).
- Turbo tasks are `build`, `dev`, `lint`, `test` (`turbo.json` lines 4-18) — the workflow will call `turbo run typecheck lint test` **after** the `typecheck` task is added (see turbo.json assignment below).
- Tests need **no Postgres** (RESEARCH: 171 tests pass with invalid `DATABASE_URL`) → no service containers.

**Shape to copy (RESEARCH Pattern 1, lines 172-192):**
```yaml
name: CI
on:
  push: { branches: [main] }
  pull_request: {}
concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true
jobs:
  verify:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4          # reads pnpm@9.15.0 from packageManager pin
      - uses: actions/setup-node@v4
        with: { node-version: 22, cache: 'pnpm' }
      - run: pnpm install --frozen-lockfile
      - run: pnpm turbo run typecheck lint test
```
> Ordering constraint: `pnpm/action-setup` MUST precede `setup-node` (so `cache: 'pnpm'` finds the store). Do NOT set `TURBO_TOKEN`/`TURBO_TEAM` (local-cache-only CI).

### `apps/api/src/index.ts` — CORS block + clerkMiddleware (config/bootstrap, request-response) — MODIFY IN PLACE

**Analog:** self. The exact block to replace is lines 24-30:
```typescript
app.use(
  cors({
    origin: process.env.FRONTEND_URL || 'http://localhost:5173',
    credentials: true,
  })
);
app.use(clerkMiddleware());
```

**Replace with** the env-driven allowlist (RESEARCH Pattern 3, lines 216-230) + `authorizedParties` on clerk (RESEARCH Pattern 4, lines 238-240):
```typescript
const staticAllow = (process.env.CORS_ALLOWED_ORIGINS ?? 'http://localhost:5173')
  .split(',').map(s => s.trim()).filter(Boolean);
const previewRe = process.env.CORS_PREVIEW_PATTERN
  ? new RegExp(process.env.CORS_PREVIEW_PATTERN)
  : null;
app.use(cors({
  origin(origin, cb) {
    if (!origin) return cb(null, true);
    if (staticAllow.includes(origin)) return cb(null, true);
    if (previewRe?.test(origin)) return cb(null, true);
    cb(new Error(`CORS: origin ${origin} not allowed`));
  },
  credentials: true,
}));
app.use(clerkMiddleware({ authorizedParties: staticAllow }));
```

**Constraints (verified in this file):**
- `import cors from 'cors'` (line 3) and `import { clerkMiddleware } from '@clerk/express'` (line 4) already present — no new imports.
- **Do NOT touch `FRONTEND_URL`** — it stays a single canonical URL (also consumed by `apps/api/src/services/email.ts` for CTA links per RESEARCH line 232). Introduce `CORS_ALLOWED_ORIGINS`/`CORS_PREVIEW_PATTERN` as *new* vars only.
- Preserve ordering: webhook router before json/cors (lines 19-20), clerk after cors — do not reorder.
- Extract `staticAllow` so it can be reused for `authorizedParties` (single source for CORS + Clerk).

**Apply to:** deployment config must set `NODE_ENV=production` (activates the guard) and the `preDeployCommand` must run **migrate only, never seed** (RESEARCH Anti-Patterns). No new code — this is a config/checklist constraint on `railway.toml` and Railway env.

**Source:** `apps/api/src/__tests__/seed/guard.test.ts` (env-driven pure fn) + `__tests__/middleware/error.test.ts` (mock res/callback capture).
**Apply to:** the new CORS origin-function test. Convention: `describe`/`it`, `vi.spyOn` + `vi.restoreAllMocks()` in `afterEach`, save/restore `process.env`, no live server or Postgres.
