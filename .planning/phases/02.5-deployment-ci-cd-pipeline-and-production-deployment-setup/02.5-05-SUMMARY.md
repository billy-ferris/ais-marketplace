## What Was Built

### `railway.toml` (repo root)
- `[build] builder = "NIXPACKS"` — explicit builder (D-10); pins away from Railway's newer Railpack default.
- `[deploy] preDeployCommand = "pnpm --filter @ais/api exec drizzle-kit migrate"` — runs in a separate container with the app image + env (incl. `DATABASE_URL`) **before** the API serves. A non-zero exit aborts the deploy and does not roll forward — the D-06 migrate-before-serve safety property. Migrate only; never `db:seed`/`db:push` (Phase 02.4 `assertNotProduction()` forbids seeding under `NODE_ENV=production`, D-05/D-08).
- `[deploy] startCommand = "pnpm --filter @ais/api exec tsx src/index.ts"` — the verified run-from-source path (NOT `node dist/index.js`). `tsx` is a promoted dependency (Plan 01), so a prod prune retains it.
- `healthcheckPath = "/api/health"` — matches the wired route (`API_ROUTES.HEALTH` in `packages/shared/src/constants/routes.ts` = `/api/health`, registered in `apps/api/src/index.ts` returning `{ status: "ok" }`).
- `restartPolicyType = "ON_FAILURE"`.

### Prerequisite facts verified in this worktree
- Root `package.json` `engines.node = "22.x"` present — Nixpacks respects it (RESEARCH A6).
- `apps/api/package.json`: `tsx` and `drizzle-kit` are in `dependencies` (not devDependencies) — survive a prod prune. `start` is `tsx src/index.ts`.
- Baseline migration exists: `apps/api/drizzle/0000_petite_scarlet_witch.sql` (+ `meta/`).
- `apps/api/drizzle.config.ts` reads `DATABASE_URL` and outputs to `./drizzle` — `drizzle-kit migrate` will apply the baseline against the plugin DB.
- `/api/health` route is present and wired.
