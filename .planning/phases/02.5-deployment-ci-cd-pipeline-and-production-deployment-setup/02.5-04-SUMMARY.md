# Phase 02.5 Plan 04: Env-Driven CORS Allowlist + Clerk authorizedParties Summary

Replaced the single-origin `cors({ origin: FRONTEND_URL })` config with an env-driven allowlist (`CORS_ALLOWED_ORIGINS`) plus an optional Vercel preview-URL regex (`CORS_PREVIEW_PATTERN`), extracted into a unit-tested helper, and bound Clerk `authorizedParties` to the same list — establishing the D-02/D-08/D-09 cross-origin security posture for the split Vercel-SPA → Railway-API deploy.

## What Was Built

**Task 1 — Testable CORS origin helper (TDD).**
`apps/api/src/lib/cors.ts` exports:
- `staticAllowedOrigins()` — parses `CORS_ALLOWED_ORIGINS ?? 'http://localhost:5173'` by comma, trims, drops empties.
- `buildCorsOptions()` — returns a `cors` options object whose `origin(origin, cb)` allows no-origin requests (curl/server-to-server/same-origin), allows any static-list member, allows a match against `CORS_PREVIEW_PATTERN` (compiled to a `RegExp` only when set), and otherwise rejects with an `Error`. `credentials: true` preserved.

`apps/api/src/__tests__/cors/origin.test.ts` (7 tests) uses the `guard.test.ts` env save/restore idiom and a callback-capture helper to prove: allowed static origin → allow; preview-pattern origin → allow; no-origin → allow; unknown origin → reject; unset preview pattern does not admit preview-shaped origins; unset `CORS_ALLOWED_ORIGINS` → localhost default; credentials stays enabled.

**Task 2 — Wired into index.ts + Clerk + env contract.**
`apps/api/src/index.ts` now uses `app.use(cors(buildCorsOptions()))` and `app.use(clerkMiddleware({ authorizedParties: staticAllowedOrigins() }))`, importing both from `./lib/cors`. Webhook-before-json and cors-before-clerk ordering preserved; no other imports changed. `FRONTEND_URL` left untouched. `apps/api/.env.example` documents `CORS_ALLOWED_ORIGINS`, `CORS_PREVIEW_PATTERN`, the distinction from `FRONTEND_URL`, and the D-08 prod notes (`NODE_ENV=production`, must NOT set `NODE_TLS_REJECT_UNAUTHORIZED`).
