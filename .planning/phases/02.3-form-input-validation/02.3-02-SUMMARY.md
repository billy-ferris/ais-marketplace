# Phase 02.3 Plan 02: Validation Timing & Required-Field Indicators Summary

Added locked validation-timing behavior (validate-on-blur, clear-on-valid-change, submit catch-all with focus) plus red required-field asterisks and live textarea character counters to all three react-hook-form manage forms, via three tiny reusable presentational helpers rather than adopting shadcn's Radix Form components.

## What Was Built

**Task 1 — FormFieldHelpers.tsx** (`1e5f782`)
New module exporting three stateless components:
- `RequiredLabel({ htmlFor, children, required })` — wraps the existing `Label`, appends a `text-destructive` asterisk (`aria-hidden`) only when `required` is truthy.
- `FieldError({ message })` — returns `null` when message is falsy, else a `text-xs text-destructive` `<p role="alert">` matching the prior inline error styling.
- `CharCounter({ value, max })` — renders `{value?.length ?? 0}/{max}` right-aligned in muted text.

Asterisks are driven exclusively by the explicit `required` prop — no runtime Zod reflection.

**Task 2 — BrandDialog + CategoryDialog** (`d88db9f`)
- Both `useForm` calls gained `mode: 'onBlur'` + `reValidateMode: 'onChange'`.
- Hand-written `{errors.x && <p/>}` blocks replaced with `<FieldError message={errors.x?.message} />`.
- Required labels converted to `RequiredLabel required`: BrandDialog name, CategoryDialog name, BrandDialog manufacturer (companyId).
- BrandDialog description Textarea gained `<CharCounter value={watch('description') ?? ''} max={2000} />`; CategoryDialog has no textarea so no counter.
- BrandDialog manufacturer Select (controlled via setValue) gained `onOpenChange` firing `trigger('companyId')` on close so an empty required select errors on blur; existing `setValue(..., { shouldValidate: true })` retained for change validation.
- Optional fields (brand description, category icon, category displayOrder) received no asterisk. React-query `isPending` button-disable left intact.

**Task 3 — ListingForm** (`ac4ee66`)
- `useForm` gained `mode: 'onBlur'` + `reValidateMode: 'onChange'`.
- Inline error blocks replaced with `FieldError`.
- Hardcoded "Name *" / "Brand *" text labels converted to `RequiredLabel required`.
- Status label intentionally left WITHOUT an asterisk and without on-blur required-validation (see Decisions).
- Description Textarea gained `<CharCounter value={watch('description') ?? ''} max={5000} />`.
- Brand Select gained `onOpenChange` firing `trigger('brandId')` on close.
- Categories section untouched (optional, managed outside RHF). Images/SKU sections untouched — SKU submit catch-all is owned by plan 02.3-03 to avoid same-wave file conflict.

## Deviations from Plan

None — plan executed exactly as written.

## Decisions Made

- **Status select gets no required treatment.** `status` has schema default `'draft'`, is pre-populated on create, and can never be emptied, so a required error can never fire. Deliberate resolved decision (02.3-VALIDATION.md), not a scope cut.
- **Listing categories get no required treatment.** `categoryIds` is optional in the schema and managed outside RHF (critical scope decision #2).
- **Controlled selects use `trigger()` on close.** Selects driven by `setValue`/`watch` (not `register`) don't auto-fire blur validation (RESEARCH Pitfall 3), so `onOpenChange(open === false)` calls `trigger(field)` to surface the required error on blur.
