## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Blur validation shows inline error on empty required field | VLDN-TIMING | Web has no test harness (out of scope) | On each form, focus then blur an empty required field → inline error appears |
| Error clears on change once valid | VLDN-TIMING | same | With an error showing, type a valid value → error clears immediately |
| Submit catch-all + focus first invalid | VLDN-TIMING | same | Leave untouched required fields empty, click Save → errors show, first invalid field is scrolled to/focused |
| SKU row submit catch-all | VLDN-SKU | same | Add a SKU row, leave a required cell (name/price/msrp/quantity) untouched, click Save → save is blocked and the row shows errors |
| Required asterisks present; optional fields have none | VLDN-INDICATORS | same | Visually confirm red `*` on required labels only |
| **Status select handling** | VLDN-INDICATORS | same — **flagged for user review** | On the CREATE form, confirm `status` is pre-set to `draft`, has NO asterisk, and cannot be emptied; confirm this handling is acceptable (intentional divergence from CONTEXT's literal "status = required select" text, because the field has a schema default and can never be empty) |
| Textarea character counters | VLDN-INDICATORS | same | Type in listing/brand description → live `x/max` counter updates |
| SKU cell red border + hover tooltip | VLDN-SKU | same | Enter an invalid price/msrp on a SKU row, blur → red border; hover → tooltip with message |
| ApiError toast on server validation failure | VLDN-APIERR | Requires live server round-trip | Trigger a server-side validation failure → toast shows the server `error` message |
