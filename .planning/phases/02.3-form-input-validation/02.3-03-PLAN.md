<objective>
Retrofit the non-RHF SkuInlineEditor with a local, blur-triggered validation layer that reuses the shared createSkuSchema — red border + Base UI Tooltip on invalid cells — WITHOUT converting the component to useFieldArray (RESEARCH anti-pattern; it has _deleted/undo/image-upload semantics that must be preserved). Also deliver the locked submit-time catch-all: when the listing form is submitted, every non-deleted SKU row is validated and an invalid row blocks the save.

Purpose: SkuInlineEditor currently has zero validation and is the largest delta in the phase. Its per-cell validation must match the main-form timing (validate on blur, clear on valid) AND enforce the CONTEXT decision "validate all fields on submit as a final catch-all" so a user cannot submit a listing with an untouched/invalid SKU row.
Output: SkuInlineEditor with local skuErrors state, blur handlers on required cells, a `validateAll()` imperative handle, tooltip-wrapped invalid inputs under a mounted TooltipProvider, and a ListingForm submit path gated on that validation.

Wave/overlap note: this plan owns the ListingForm submit-wiring seam. Because ListingForm is also edited by plan 02.3-02 (Wave 1), this plan runs in Wave 2 and depends_on 02.3-02, so the two plans never touch ListingForm.tsx concurrently.
</objective>
