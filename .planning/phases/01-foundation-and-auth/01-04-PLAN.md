  <action>
Implement Clerk-based authentication and authorization for the Express API.

1. **apps/api/src/types/globals.d.ts** — Declare custom session claims type:
   ```typescript
   declare global {
     interface CustomJwtSessionClaims {
       metadata: {
         role?: import('@ais/shared').UserRole;
       };
     }
   }
   export {};
   ```
   This tells TypeScript about the custom `metadata` field in Clerk session tokens (configured in Clerk Dashboard to include `{{user.public_metadata}}`).

2. **apps/api/src/middleware/auth.ts**:
   - Re-export `clerkMiddleware` and `requireAuth` from `@clerk/express`
   - Create `requireRole(...roles: UserRole[]): RequestHandler`:
     - Call `getAuth(req)` to get auth context
     - Read `auth.sessionClaims?.metadata?.role`
     - If role not present or not in allowed roles, return 403 `{ error: 'Forbidden' }`
     - Otherwise call `next()`
   - Create `getCurrentUser(req): { userId, role, sessionClaims }` helper that extracts auth info

3. **apps/api/src/routes/webhooks.ts**:
   - Create Express Router
   - CRITICAL: Use `express.raw({ type: 'application/json' })` as route-level middleware (NOT express.json — per pitfall #1 from research)
   - POST `/clerk` handler:
     - Instantiate `new Webhook(process.env.CLERK_WEBHOOK_SIGNING_SECRET!)`
     - Extract svix headers: `svix-id`, `svix-timestamp`, `svix-signature`
     - Call `wh.verify(req.body, headers)` — if it throws, return 400
     - Handle event types:
       - `user.created`: Upsert into local `users` table with clerkId, email, firstName, lastName, role from publicMetadata (default 'retailer'), companyId null (will be set by seed or admin later)
       - `user.updated`: Update local user row by clerkId
       - `user.deleted`: Delete local user row by clerkId
     - Return 200 `{ received: true }`

4. **Update apps/api/src/routes/companies.ts**:
   - Add `requireAuth()` and `requireRole('admin')` middleware to POST and PATCH routes (only admin can create/modify companies)
   - GET routes remain accessible to any authenticated user (add `requireAuth()` only)

5. **Update apps/api/src/routes/users.ts**:
   - Add `requireAuth()` to all routes
   - Add `requireRole('admin')` to GET list (only admin sees all users)
   - Add GET `/me` route that returns the current user's data from local DB (lookup by clerkId from `getAuth(req).userId`)

6. **Update apps/api/src/index.ts**:
   - Ensure webhook router is mounted BEFORE `express.json()` middleware
   - Mount webhook router at `/api/webhooks` using the webhookRouter
   - All other API routes come after `express.json()`, `cors()`, and `clerkMiddleware()`
