<!-- Listing route handler source -->

From apps/api/src/routes/listings.ts (exported as listingRouter):

GET / -- List listings with pagination. Filters: search (ilike name), status (eq), brandId (eq). Excludes soft-deleted. Uses subquery for SKU count. Uses Promise.all for [data, [{total}]]. After data query, fetches categories for returned listing IDs in a separate query. Returns enrichedData with categories array per listing.

GET /:id -- Get full listing with relations via db.query.brandListings.findFirst() with `with` clause: brand (with company), inventorySkus (filtered by not deleted), brandListingImages (ordered by displayOrder), listingCategories (with category). Filters deleted categories. Returns 404 if not found.

POST / -- Create listing with nested entities. Body: { listing, skus, images, categoryIds }. Parses listing with createListingSchema. Inserts listing, then SKUs (if provided, parsed with createSkuSchema), images (max 5, ensures one isPrimary), categories (join table). Re-queries with full relations. Returns 201.

PATCH /:id -- Update listing with granular operations. Body: { listing, skus: { create, update, delete }, images: { create, delete }, categoryIds }. Checks listing exists. Updates listing fields if provided. Creates/updates/soft-deletes SKUs. Creates images (enforces max 5 total), hard-deletes images. Replaces categories if categoryIds provided. Re-queries with full relations.

DELETE /:id -- Soft delete listing. Sets deletedAt on listing. Cascade: soft-deletes all SKUs (sets deletedAt), hard-deletes listing_categories rows, hard-deletes brand_listing_images rows. Returns { message: 'Listing deleted' }.

Export: `export { router as listingRouter }`
