Create apps/api/src/routes/listings.ts following the established CRUD route pattern (Router with explicit RouterType, requireAuth + requireRole('admin'), try/catch with next(err)):

**GET / -- List listings:**
- Filter: isNull(brandListings.deletedAt).
- Support query params: search (ilike on brandListings.name), status (eq filter on status column), brandId (eq filter), page (default 1), limit (default 20).
- Join with brands to include brand name in response.
- For each listing, include a count of non-deleted inventory_skus (SKU count) via a subquery or separate query. Also include the listing's categories (join through listing_categories to categories, filter isNull(categories.deletedAt)).
- Return { data: listings[], total: number, page: number, limit: number }.

**GET /:id -- Get listing by ID:**
- Filter: not deleted, eq id.
- Include full relations: brand (with company), inventorySkus (non-deleted), brandListingImages (ordered by displayOrder), categories (non-deleted, via listing_categories join).
- Return the full listing object or 404.

**POST / -- Create listing:**
- Accept body: { listing: CreateListingInput, skus?: CreateSkuInput[], images?: { imageUrl: string, displayOrder: number, isPrimary: boolean }[], categoryIds?: number[] }.
- Parse listing fields with createListingSchema.
- Insert brand_listings row. Get the returned listing ID.
- If skus provided: validate each with createSkuSchema, insert into inventory_skus with the listing ID.
- If images provided: insert into brand_listing_images with listing ID. Enforce max 5 images. Ensure exactly one isPrimary (if none flagged, set first as primary).
- If categoryIds provided: insert into listing_categories join table.
- Return 201 with the full listing (re-query with all relations like GET /:id).

**PATCH /:id -- Update listing:**
- Accept body: { listing?: UpdateListingInput, skus?: { create?: CreateSkuInput[], update?: { id: number, data: UpdateSkuInput }[], delete?: number[] }, images?: { create?: { imageUrl: string, displayOrder: number, isPrimary: boolean }[], delete?: number[] }, categoryIds?: number[] }.
- If listing fields provided: parse with updateListingSchema, update brand_listings row with updatedAt.
- If skus.create: insert new SKUs for this listing.
- If skus.update: for each, parse with updateSkuSchema, update inventory_skus row.
- If skus.delete: soft-delete the specified SKU IDs (set deletedAt).
- If images.create: insert new images (enforce max 5 total non-deleted).
- If images.delete: hard-delete image rows (images are not soft-deleted since they're just URL references).
- If categoryIds provided: delete all existing listing_categories for this listing, re-insert the new set (replace strategy).
- Return updated listing with full relations or 404.

**DELETE /:id -- Soft delete listing:**
- Set deletedAt on brand_listings row.
- Also soft-delete all inventory_skus belonging to this listing (cascade soft delete).
- Hard-delete listing_categories rows for this listing (join table cleanup).
- Hard-delete brand_listing_images rows for this listing (image refs cleanup).
- Return deleted listing or 404.

**Register in apps/api/src/index.ts:** Add `app.use(API_ROUTES.LISTINGS, listingRouter)` after existing route mounts. Import listingRouter.

IMPORTANT: All list queries must filter isNull(deletedAt). Use a helper like `const notDeleted = (table: any) => isNull(table.deletedAt)` at the top of the file for consistency. For the GET /:id response, use Drizzle's relational query API (db.query.brandListings.findFirst with `with` option) to load nested relations in one call.

- `useListings(params?: { search?: string; status?: string; brandId?: number; page?: number })` -- queryKey ['listings', 'list', params].
- `useListing(id: number)` -- queryKey ['listings', 'detail', id]. Enabled when id is truthy. Returns full listing with SKUs, images, categories.
- `useCreateListing()` -- mutation POSTing to API_ROUTES.LISTINGS. Body includes { listing, skus, images, categoryIds }. onSuccess: invalidate ['listings'], toast.success, navigate to /manage/listings.
- `useUpdateListing()` -- mutation PATCHing to API_ROUTES.LISTINGS/:id. onSuccess: invalidate ['listings'], toast.success.
- `useDeleteListing()` -- mutation DELETEing. onSuccess: invalidate ['listings'], toast.success.
