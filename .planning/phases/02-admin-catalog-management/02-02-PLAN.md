**Brand API route (apps/api/src/routes/brands.ts):** Follow the exact pattern from companies.ts with `const router: RouterType = Router()` and try/catch with next(err).

- GET / -- List brands. requireAuth() + requireRole('admin'). Filter: isNull(brands.deletedAt). Support query params: search (ilike on brands.name), companyId (eq filter), page (default 1), limit (default 20). Return array of brands with pagination metadata (total count via separate count query). Join with companies to include company name in response.
- GET /:id -- Get brand by ID. requireAuth() + requireRole('admin'). Filter: isNull(brands.deletedAt), eq(brands.id, id). Include company relation. Return 404 if not found.
- POST / -- Create brand. requireAuth() + requireRole('admin'). Parse body with createBrandSchema from @ais/shared/schemas. Auto-generate slug from name (lowercase, replace spaces/special chars with hyphens, trim). If slug exists, append -2, -3, etc. Insert and return 201.
- PATCH /:id -- Update brand. requireAuth() + requireRole('admin'). Parse body with updateBrandSchema. If name provided, regenerate slug (with uniqueness check). Set updatedAt to new Date(). Return 404 if not found or deleted.
- DELETE /:id -- Soft delete brand. requireAuth() + requireRole('admin'). Set deletedAt to new Date(). Return 404 if not found or already deleted.

**Category API route (apps/api/src/routes/categories.ts):** Same pattern.

- GET / -- List categories. requireAuth() + requireRole('admin'). Filter: isNull(categories.deletedAt). Support: search (ilike name), page, limit. Order by displayOrder ASC. Return with total count.
- GET /:id -- Get category by ID. Filter: not deleted. Return 404 if not found.
- POST / -- Create category. Parse with createCategorySchema. Auto-generate slug from name. Insert and return 201.
- PATCH /:id -- Update. If name provided, regenerate slug. Set updatedAt.
- DELETE /:id -- Soft delete with deletedAt.

**Upload API route (apps/api/src/routes/uploads.ts):**

- Create S3Client configured for R2: region 'auto', endpoint from R2_ACCOUNT_ID env var, credentials from R2_ACCESS_KEY_ID and R2_SECRET_ACCESS_KEY env vars.
- POST /presigned-url -- requireAuth() + requireRole('admin'). Accept {fileName, contentType} in body. Generate unique key: `images/${Date.now()}-${sanitizedFileName}`. Create PutObjectCommand with Bucket (R2_BUCKET_NAME), Key, ContentType. Use getSignedUrl with 600s (10 min) expiry. Return { uploadUrl, publicUrl: `${R2_PUBLIC_URL}/${key}`, key }.
- PITFALL: ContentType in PutObjectCommand MUST match the Content-Type header the browser sends. Always use the contentType from the request body.

**Register routes in apps/api/src/index.ts:** Add `app.use(API_ROUTES.BRANDS, brandRouter)`, `app.use(API_ROUTES.CATEGORIES, categoryRouter)`, `app.use(API_ROUTES.UPLOADS, uploadRouter)` after existing route mounts. Import from new route files.

**Update apps/api/.env.example:** Add R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_BUCKET_NAME, R2_PUBLIC_URL entries.

- Loading state: show Skeleton rows when isLoading is true.
- Empty state: show "No results found" when data is empty and not loading.

**TanStack Query hooks:**

**useBrands.ts:** Create hooks following the TanStack Query v5 pattern from RESEARCH.md:
- `useBrands(params?: { search?: string; page?: number; limit?: number })` -- useQuery with queryKey ['brands', 'list', params]. Calls apiFetch with query string params. Returns brand list + pagination.
- `useBrand(id: number)` -- useQuery with queryKey ['brands', 'detail', id]. Enabled only when id is truthy.
- `useCreateBrand()` -- useMutation calling POST to API_ROUTES.BRANDS. onSuccess: invalidate queryKey ['brands'] and show toast.success('Brand created').
- `useUpdateBrand()` -- useMutation calling PATCH to API_ROUTES.BRANDS/:id. onSuccess: invalidate ['brands'] and toast.success('Brand updated').
- `useDeleteBrand()` -- useMutation calling DELETE to API_ROUTES.BRANDS/:id. onSuccess: invalidate ['brands'] and toast.success('Brand deleted').
Import toast from sonner.

**useCategories.ts:** Same pattern:
- `useCategories(params?)` -- queryKey ['categories', 'list', params].
- `useCategory(id)` -- queryKey ['categories', 'detail', id].
- `useCreateCategory()`, `useUpdateCategory()`, `useDeleteCategory()` -- mutations with invalidation and toast.

**useUpload.ts:** Create upload hook:
- `useUpload()` returns `{ uploadFile, isUploading }`.
- `uploadFile(file: File)` is an async function that: (1) calls apiFetch POST to `${API_ROUTES.UPLOADS}/presigned-url` with { fileName: file.name, contentType: file.type }, (2) does a fetch PUT to the returned uploadUrl with file body and Content-Type header matching file.type, (3) returns the publicUrl string.
- Track isUploading state via internal useState.

**BrandDialog.tsx:** shadcn Dialog with react-hook-form + zodResolver:
- Props: open, onOpenChange, brand? (for edit mode, optional).
- Form fields: name (Input), description (Textarea), companyId (Select populated by fetching companies list via apiFetch -- GET /api/companies returns all companies, filter to type='manufacturer' on client), logoUrl (ImageUploader component).
- For logo: when user selects an image in ImageUploader, call uploadFile() from useUpload, then set logoUrl field value to the returned publicUrl.
- On submit: call useCreateBrand or useUpdateBrand mutation depending on whether brand prop exists.
- On success: close dialog, form resets.
- Submit button shows loading state when mutation is pending.
- Dialog title: "Create Brand" or "Edit Brand" based on mode.

**CategoryDialog.tsx:** simpler dialog:
- Props: open, onOpenChange, category? (for edit mode).
- Form fields: name (Input), icon (Input with helper text "Enter a Lucide icon name, e.g., scissors, sparkles"), displayOrder (Input type number).
- On submit: useCreateCategory or useUpdateCategory.
- On success: close dialog, form resets.

**brand-columns.tsx:** TanStack Table column definitions for brands:
